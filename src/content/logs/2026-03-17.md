3 月 17 日的节奏很紧，几条线都不是从零起步，而是在把已经露出形状的东西继续收口。最明显的四条主线分别是博客内容结构继续拆细、Daily Scaffold 的发布链路补齐、`paper-form-ai-filling` 从测试资产走到回归基线，以及 `fachi-web` 正式接进发布体系并推到 production。

## 早上

早上先把博客和日志系统再收紧一轮。`myport` 把站点正式拆成 `articles` 和 `logs` 两条内容线，首页、总入口、栏目页、详情页和 RSS 一起跟着调整，最后用 `2224bf2 Publish articles and logs sections` 收口。紧接着又补齐并发布了 3 月 12 日到 15 日的日志，把前几天已经形成的整理结果真正接进公开内容流里。

同一时段，`daily-scaffold-review` 也继续被收简。之前那些偏模板化、偏文风参考的部分被进一步拿掉，只留下日期窗口、来源优先级、`verified / inference` 区分和 sidecar 落盘规则，主文档标准被压到一句话: 完整且精炼。对这条线来说，重点不是多写一种格式，而是把私有底稿和公开日志之间的边界收得更清楚。

## 白天

上午后半段到下午，`paper-form-ai-filling` 成了最重的一条工程线。先从本地 `dev-assets` 里整理出一套可复用的数据集，`checklist.json` 固定下 10 条样本和对应资源，后面的回归就不再靠临时挑案例。接着在 `lvshe-be` 里补 live regression 脚本、回归测试和辅助工具，把测试资产、执行脚本和验证入口串成一条能反复跑的链路。

这条线当天最关键的进展，不是“又加了几个测试文件”，而是两个长期卡点终于被打通。`execution-objection-procedural-elements` 不再因为 JSON 截断失败，`house-lease-commercial-operation-dispute` 也不再因为 token limit 直接拒绝。到下午重跑 `doubao 2.0 mini` 时，9 个 active case 已经能完整打分，不过最难的几条 fixture 还没有真正收口，后面仍要继续看 gold 和输出之间的差异。

## 下午到晚上

中午到下午，`boomyao.github.io` 本身又长出了一条新内容线: 小记。站点新增 `notes` collection、列表页和详情页，首批落了《先把灵感留下来》和《过去那些理想化的方案，在现在开始逐个落地》两篇内容，对应发布提交是 `ccbbdb1`。随后又把 GitHub Actions 调到 Node 24 兼容版本，相关 action 升级后以 `86b3491` 推到 `origin/master`，当轮 `CI` 和 `Deploy` 都顺利通过。

傍晚以后，`fachi` 这条线闭环得最完整。`lvshe-fe` 新增 `fachi-web` 官方站发布链路并修掉前端异常，`lvshe_infra` 同步补上 service 部署和 production overlay，最后把 production 版本推进到 `0.20260317.2`。同一晚还补跑了多轮只执行不改码的 live regression，横向比较不同模型的表现。连接层已经没有明显问题，但难例仍集中在 `construction-contract-project-payment`、`house-lease-commercial-operation-dispute` 和 `sale-contract-prepayment-guarantee-capital-contribution`，说明 token 上限和 JSON 稳定性依旧会被模型差异重新放大。

## 收口

这一天最值得记住的，不是某一个单点产物，而是几条线都在做同一件事: 把结构固定下来，换取后续复用的确定性。博客从文章、日志继续延伸到小记，Daily Scaffold 的私有整理标准继续收紧，`paper-form-ai-filling` 从零散实验推进到有数据集、有脚本、有难例清单的阶段，`fachi-web` 则直接完成了从链路接入到 production 发布的闭环。还没关掉的口也很清楚，`paper-form-ai-filling` 最难的几条 fixture 还要继续压，晚上的多模型回归也说明模型选择这件事远没到可以拍板的时候。
