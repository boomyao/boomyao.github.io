这一天表面上摊得很开，实际上有一条很清楚的暗线在牵着我走：我在重新设计自己怎么记录、怎么验证、怎么把事情推到能长期复用的状态。Claude、ChatGPT 和远端 Codex 都没给出当天可用正文，所以这篇日志主要建立在本机 Codex 和 repo 产物上；也正因为来源集中，反而能看清楚我那天反复回到的不是某一个功能点，而是几种长期摩擦本身。

早上先动的不是业务功能，而是记录系统。`myport` 里把博客拆成 `articles` 和 `logs` 两条内容线，首页、`/blog` 总入口、栏目页、详情页和 RSS 一起调整；紧接着又去补 `boomyao.github.io` 上前几天缺着的日志，再把 `2026-03-16` 发到 `origin/main`。如果只看文件，这像一次内容站改版；但把几段 session 放在一起看，它更像我对“日常产出该怎么留下来”这件事继续不满意，所以不再满足于往旧结构里塞内容，而是开始改内容结构本身。

这种不满意很快又回到了 Daily Scaffold 自己身上。上午那几段 Codex 会话里，我一边让它输出“完整但精炼”的昨日日志，一边继续删 `daily-scaffold-review` 里的 `output template` 和风格参考。这里真正被我盯上的不是一篇日志，而是生成日志的工作流。我想要的已经不是“格式更像日志”，而是让它别再靠最后一次汇总硬压出成品。这一点放到现在回看很关键，因为它解释了为什么同一天我后来会自然接受“小记”这种新的内容形态，也解释了为什么我会直接对旧的 scaffold / meta 二分法失去耐心。

上午后半段到下午，注意力就转到了另一种长期摩擦：`paper-form-ai-filling` 到底怎么才能被稳定地测。先是在 `lvsheyun-devtools` 里把 `paper-form-ai-filling` 的数据集和 `checklist.json` 整理出来，等于先回答“拿什么测”；随后到 `lvshe-be` 里补 live regression 脚本、回归测试、辅助工具，再去改 `paper-form-ai-filling.ai.ts` 和 `fetch-doubao.ts`，等于继续回答“怎么让它可重复地测”。这条线最像工程上的我：不是先追着一个失败样本打补丁，而是先把样本、脚手架、模型输入、返回兼容和对比方式都搭出来，让“模型到底稳不稳”从一种感觉变成可以反复执行、反复比较的事情。

中间插进来的 `boomyao.github.io` 小记模块，也不是枝节。两篇首批小记一篇叫《先把灵感留下来》，一篇叫《过去那些理想化的方案，在现在开始逐个落地》。把它们放回这一天的上下文里看，意思很直白：文章和日志都已经不够承接我当天真正想留下来的东西了，于是我又给“还没长成文章、也不该塞进日终复盘的念头”单独开了一条通道。随后又顺手把 CI / Deploy 的 Node 24 兼容问题收掉，这一步也很像同一套习惯的延伸：新结构刚搭出来，就立刻把它的发布摩擦一起压掉，不让它只停在本地可用。

傍晚之后，`fachi-web` 那条线把“长期结构”这件事推到了最外层。它不是单纯把一个官网项目拷进仓库，而是一路接到前端发布流水线、基础设施 service 部署，再推进到 production `0.20260317.2`。如果说白天我更多是在处理“怎么记录自己”和“怎么验证模型”，那到了晚上，我处理的就是“怎么把已经做出来的东西稳定推到线上”。同一晚继续跑多轮只执行不改码的 live regression 也说明了这一点：我没有急着把模型结果讲成一次成功，而是愿意把最难看的几条 fixture 摆在桌上，承认 `construction-contract-project-payment`、`house-lease-commercial-operation-dispute`、`sale-contract-prepayment-guarantee-capital-contribution` 这些地方还在反复暴露 token 和 JSON 稳定性问题。

所以如果一定要给 3 月 17 日下一个判断，我会说：这不是一个“多项目并行”的日子，而是一个“我在给自己未来会反复做的事情重新修路”的日子。记录系统上，文章 / 日志 / 小记开始分层；验证系统上，`paper-form-ai-filling` 从样本到回归开始成形；发布系统上，`fachi-web` 从接入走到了 production。真正还没收住的口，也都还是这些路上的口：Daily Scaffold 生成还没彻底摆脱一次性汇总，`paper-form-ai-filling` 的难例还要继续看 gold / 输出对比，ChatGPT 和远端 Codex 这两条来源还被 DNS 卡着。它们没有破坏当天的主线，反而把主线衬得更清楚了：我不是在堆任务，我是在一点点把“以后怎么工作”改得更顺手。
